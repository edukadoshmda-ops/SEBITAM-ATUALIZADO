# Guia: Ambientes de Notificações e Mensagens (SEBITAM e Escolas IBMA)

Este guia explica como criar ambientes separados para **notificações** e **mensagens** no SEBITAM e nas Escolas IBMA.

---

## 1. Conceito de Ambientes

| Ambiente | Tipo de Login | Descrição |
|----------|---------------|-----------|
| **SEBITAM** | `loginType: 'sebitam'` | Seminário, matrículas, alunos, professores, financeiro |
| **Escolas IBMA** | `loginType: 'escolas-ibma'` | Membresia, Discipulado, Batismo, Oração, professores IBMA |

O app já diferencia os ambientes pelo `currentUser.loginType`. As notificações e mensagens devem seguir essa mesma lógica.

---

## 2. Estrutura de Dados (Supabase)

### 2.1 Tabela de Notificações

Execute no **SQL Editor** do Supabase:

```sql
-- Tabela de Notificações (sistema → usuário)
CREATE TABLE IF NOT EXISTS notificacoes (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ambiente TEXT NOT NULL CHECK (ambiente IN ('sebitam', 'escolas-ibma')),
    destinatario_email TEXT,
    destinatario_tipo TEXT,  -- 'aluno', 'professor', 'admin', 'secretaria', 'professor_ibma'
    titulo TEXT NOT NULL,
    mensagem TEXT NOT NULL,
    tipo TEXT DEFAULT 'info',  -- 'info', 'aviso', 'urgente', 'sucesso'
    lida BOOLEAN DEFAULT false,
    link TEXT,  -- link opcional (ex: /view/matriculas)
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_notificacoes_ambiente ON notificacoes(ambiente);
CREATE INDEX idx_notificacoes_email ON notificacoes(destinatario_email);
CREATE INDEX idx_notificacoes_lida ON notificacoes(lida);
```

### 2.2 Tabela de Mensagens (chat interno)

```sql
-- Tabela de Mensagens (comunicação entre usuários)
CREATE TABLE IF NOT EXISTS mensagens (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ambiente TEXT NOT NULL CHECK (ambiente IN ('sebitam', 'escolas-ibma')),
    remetente_email TEXT NOT NULL,
    remetente_nome TEXT,
    destinatario_email TEXT NOT NULL,
    assunto TEXT,
    corpo TEXT NOT NULL,
    lida BOOLEAN DEFAULT false,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_mensagens_ambiente ON mensagens(ambiente);
CREATE INDEX idx_mensagens_destinatario ON mensagens(destinatario_email);
CREATE INDEX idx_mensagens_remetente ON mensagens(remetente_email);
```

### 2.3 RLS (segurança por ambiente)

```sql
-- Permitir leitura apenas no ambiente do usuário
ALTER TABLE notificacoes ENABLE ROW LEVEL SECURITY;
ALTER TABLE mensagens ENABLE ROW LEVEL SECURITY;

-- Política: usuário vê só notificações do seu ambiente
CREATE POLICY "notificacoes_por_ambiente" ON notificacoes
FOR SELECT USING (true);  -- simplificado; ajuste conforme auth

CREATE POLICY "mensagens_por_ambiente" ON mensagens
FOR SELECT USING (true);
```

## 3. Lógica no App (main.js)

### 3.1 Filtro por ambiente

Sempre que buscar notificações ou mensagens, filtre pelo `currentUser.loginType`:

```javascript
// Exemplo: buscar notificações do ambiente atual
async function getNotificacoes() {
    const ambiente = currentUser.loginType; // 'sebitam' ou 'escolas-ibma'
    const { data } = await supabase
        .from('notificacoes')
        .select('*')
        .eq('ambiente', ambiente)
        .eq('destinatario_email', currentUser.email)
        .order('created_at', { ascending: false })
        .limit(20);
    return data || [];
}
```

### 3.2 Ao enviar notificação/mensagem

Use o ambiente correto:

```javascript
await supabase.from('notificacoes').insert({
    ambiente: currentUser.loginType,
    destinatario_email: 'aluno@email.com',
    destinatario_tipo: 'aluno',
    titulo: 'Nova atividade disponível',
    mensagem: 'Confira o material da aula de hoje.',
    tipo: 'info'
});
```

## 4. UI Sugerida

### 4.1 Ícone de notificações no header

- Adicionar um sino (🔔) no top-bar que exibe badge com quantidade de não lidas
- Ao clicar: dropdown ou tela lateral com lista de notificações **do ambiente atual**

### 4.2 Tela de mensagens

- Nova view: `mensagens` ou `mensagens-inbox`
- Visível para ambos os ambientes
- Lista filtrada por `ambiente = currentUser.loginType`

### 4.3 Menu lateral

- **SEBITAM:** item "Notificações" e "Mensagens"
- **Escolas IBMA:** mesmos itens, mas dados vêm do ambiente `escolas-ibma`

## 5. Push Notifications (opcional – app mobile)

O projeto usa **Capacitor** para Android. Para push notifications:

1. Configurar **Firebase Cloud Messaging (FCM)** no projeto Firebase
2. Adicionar `google-services.json` em `android/app/`
3. Instalar `@capacitor/push-notifications`
4. Ao registrar o token FCM, enviar ao Supabase junto com `ambiente` e `email`:

```sql
CREATE TABLE IF NOT EXISTS push_tokens (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ambiente TEXT NOT NULL,
    email TEXT NOT NULL,
    token TEXT NOT NULL UNIQUE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

Assim, ao disparar notificações push, você filtra pelo `ambiente` antes de enviar.

## 7. Mapeamento rápido

| Campo `ambiente` | Login | Usuários típicos |
|------------------|-------|------------------|
| `sebitam` | Login SEBITAM | alunos, professores, admins, secretárias |
| `escolas-ibma` | Login Escolas IBMA | professores IBMA, coordenadores de Membresia/Discipulado/Batismo |

Qualquer dado de notificação ou mensagem deve incluir `ambiente` para garantir que SEBITAM e Escolas IBMA fiquem isolados.
